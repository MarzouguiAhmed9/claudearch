- **Effort:** S
- **Priority:** P0

### 2. Token collisions at 32 bits — silent depseudo corruption

- **Where:** `backend/privacy_proxy/app/pseudonymizer.py:55`
- **Why:** `hashlib.sha256(value.encode()).hexdigest()[:8]` = 32 bits. Birthday collision at ~65k entities. `mapping[token] = original` (pseudonymizer.py:215) silently overwrites on collision, so depseudo returns the wrong original. In a Redis-backed multi-user proxy this is an actual data leak risk over time.
- **Fix:**
  ```py
  def generate_token(entity_type: str, value: str) -> str:
      h = hashlib.sha256(value.encode()).hexdigest()[:16]  # 64 bits
      return f"{entity_type}_{h}"
  ```
  Also assert-on-collision so bugs are loud, not silent:
  ```py
  # in pseudonymize()
  existing = mapping.get(token)
  if existing is not None and existing != original:
      raise RuntimeError(f"token collision: {token} → {existing!r} vs {original!r}")
  ```
- **Effort:** S
- **Priority:** P0

### 3. Redis stores plaintext PII mappings + swallows write errors

- **Where:** `backend/privacy_proxy/app/mapping_store.py:83, 53-59, 85-86`
- **Why:** `json.dumps(mapping)` written raw. Anyone with Redis access reads every pseudonym → real-name mapping the proxy has ever handled. And `except Exception` on Redis write means a failed write is only `print()`ed — depseudo will later return the pseudonymized text as-is (user sees `PERSON_abc123` instead of "Max"), or worse, the depseudo happens from stale memory that got evicted differently across replicas.
- **Fix (minimum, ship first):**
  - Encrypt values with a per-deploy symmetric key from env (`MAPPING_ENCRYPTION_KEY`) using `cryptography.fernet.Fernet` — 10 lines.
  - Convert `except Exception` to explicit `redis.exceptions.RedisError`, and surface it in `log_error()` not `print()`.
- **Fix (proper, later):** move mapping store into the vHSM (Enclaive tech) — see the enclaivetech skill for how.
- **Effort:** M
- **Priority:** P0

### 4. check_leakage() case-sensitivity bug

- **Where:** `backend/privacy_proxy/app/depseudonymizer.py:12-18`
- **Why:** `if pattern in text.lower()` finds the pattern, then `if original in text` scans original-case text. If the LLM writes "Refers to Max Mustermann" (title case), the pattern hit fires but the `original in text` check works, so this specific case is OK — BUT `text.lower()` isn't cached, and `mapping.values()` scan is O(n×m) inside a `for pattern in LEAK_PATTERNS`. Real issue: LEAK_PATTERNS is English-only (7 phrases), production has German too — leak-through undetected. Also once a pattern hits, we replace *every* original in text, even ones unrelated to the leak. Aggressive but arguably safe.
- **Fix:**
  - Add German patterns: `"bezieht sich auf"`, `"steht für"`, `"heißt eigentlich"`, `"das ist"`, `"bekannt als"`.
  - Cache `text_lower = text.lower()` outside the loop.
  - Break the outer loop on first hit — no reason to keep scanning after redaction.
- **Effort:** S
- **Priority:** P0

### 5. MappingStore memory leak — dead sessions never expire

- **Where:** `backend/privacy_proxy/app/mapping_store.py:33-39`
- **Why:** `_cleanup(session_id)` only runs when *that specific* session_id is accessed. Sessions the user abandons stay in `self._store` and `self._timestamps` forever. Under long-running load (weeks in prod, k8s pod uptime) the proxy grows unbounded → OOMKilled. Especially bad for the k8s migration.
- **Fix (lazy, ship first):**
  ```py
  # ponytail: sweep all expired sessions on any access. O(n) but n is small
  # in practice. Move to background asyncio task if n grows past ~10k.
  def _cleanup(self, session_id: str):
      now = time.time()
      expired = [sid for sid, ts in self._timestamps.items() if now - ts > self._ttl]
      for sid in expired:
          self._store.pop(sid, None)
          self._timestamps.pop(sid, None)
      self._timestamps[session_id] = now
  ```
- **Effort:** S
- **Priority:** P0

### 6. Session ID fallback = MD5(first_msg) → cross-user session bleed

- **Where:** `backend/privacy_proxy/app/main.py:369-373`
- **Why:** When `chat_id` is missing, session id = `hashlib.md5(first_msg.encode()).hexdigest()[:12]`. Two different users whose first message happens to be the same string ("Hi", "test", "hello") share a mapping. User A's PII gets depseudonymized in user B's chat.
- **Fix:** include a per-request/per-connection identifier in the fallback — the OWU user id from the JWT if present, otherwise `secrets.token_hex(6)` so at least it's unique per request (loses cross-message continuity, but that's better than cross-user leakage).
  ```py
  import secrets
  session_id = (
      body.get("chat_id")
      or (messages[0].get("id") if messages else None)
      or secrets.token_hex(6)
  )
  ```
- **Effort:** S
- **Priority:** P0

### 8. Sorting mapping keys inside streaming loop

- **Where:** `backend/privacy_proxy/app/main.py:208, 217`
- **Why:** For every safe chunk yielded, we call `sorted(mapping.keys(), key=len, reverse=True)`. If the LLM streams 500 chunks and the mapping has 20 tokens, that's 500 × O(20 log 20) sorts. Sort once outside the loop.
- **Fix:**
  ```py
  sorted_tokens = sorted(mapping.keys(), key=len, reverse=True)
  # ... use sorted_tokens inside the async for
  ```
- **Effort:** S
- **Priority:** P1

### 9. _TrackedDict.__setitem__ copies dict on every entity write

- **Where:** `backend/privacy_proxy/app/mapping_store.py:109-111`
- **Why:** `self._store._memory[self._session_id] = dict(self)` — a full copy on every entity insertion. It's also redundant: `_TrackedDict` inherits from `dict`, and `_memory[session_id]` already points at `self` since `setdefault` stored the initial `value` object. Any mutation is already visible.
- **Fix:** Delete lines 110-111. Trust the reference.
  ```py
  def __setitem__(self, key, value):
      super().__setitem__(key, value)
  ```
  Add one-line assert self-check to prove reference identity survives, then delete.
- **Effort:** S
- **Priority:** P1

### 12. bare except: swallows all errors

- **Where:** `pseudonymizer.py:102, 203`; `mapping_store.py:13, 58, 85, 96`; `main.py:158, 197`
- **Why:** Real bugs hide behind `except:` / `except Exception:`. Every one of these should be narrow (`LangDetectException`, `RedisError`, `httpx.HTTPError`, `orjson.JSONDecodeError`). Right now a `KeyError` in the analyzer would be silently caught and default to `"en"`.
- **Fix:** Narrow each except clause to the actual expected exception type. If unsure, `logging.exception()` and re-raise for the unexpected case.
- **Effort:** M
- **Priority:** P1

### 13. `expand_query` uses print() instead of structured logs

Also — since this hits gpt-4o-mini directly, if `expand_query` is called with the user's raw pre-pseudonymized question (need to verify call-site ordering), it's leaking PII to a second LLM endpoint.

### 17. check_leakage() double-replaces on multiple pattern hits

- **Why:** If two patterns hit ("refers to" and "means"), we scan `mapping.values()` twice, doing `text.replace()` on originals that were already redacted on the first pass. Not a bug (the second replace is a no-op on redacted `[REDACTED]`), just wasted CPU. `break` after first hit.
